为什么选择FIS(Winson如是说)

  • 配置简单，使用简单
  • 直接面向工程
  • 中文资料多
  • 大公司支撑，稳定可靠
  • 易于拓展

FIS3之于FIS2

FIS3进一步简化了配置，语义化更好，可配置性更高，并且支持链式语法

安装

  • FIS3 的构建不会修改源码，而是会通过用户设置，将构建结果输出到指定的目录
    • 源码 —— src
    • 编译规则 —— fis-conf.js
    • 目标目录 —— dist
  • Windows下安装
    • npm install -g fis3

构建及调试(一)

  • 命令
    • fis3 server open —— FIS3内置Server目录
    • fis3 release —— 发布
    • fis3 server start —— 启动
    • -h —— 命令行帮助
    • -wL —— 监听及自动刷新

FIS3语法规则


    fis.media('W20E')
    .match('**.{scss,css}', {
        useHash: true,
        optimizer: 'sass'//或optimizer: fis.plugin('sass', {outputStyle: 'expanded'})
    })
    .match('**/*ipcom.{scss,css}', {
        release: false
    })

    command:

    fis3 release W20E -d ../dist

Reasy之于FIS3

  • Reasy之于FIS3(针对于Tenda业务基于FIS3进行了封装)
    • 更易安装 — reasy-install.bat
    • 命令更精简 — reasy命令即可完成基本编译 支持右键自动编译
    • 配置更精简 — 对于一些插件的配置进行了封装，配置更简洁
    • 插件更可靠 — 持保留意见 ⊙﹏⊙‖∣
    • 更利于团队部署 — 举例：微企W20E系列，便携A9系列
